# 敏感数据类型及其规范脱敏规则（符合GB/T 45574-2025）

## 概述

本文档依据国家标准GB/T 45574-2025《数据安全技术 敏感个人信息处理安全要求》，详细定义了各种敏感数据类型的脱敏规则。该标准于2025年11月1日正式实施，旨在规范敏感个人信息处理活动，保障个人信息主体权益。

## 1. 姓名脱敏规则

### 1.1 中文姓名
- **规则**: 保留姓氏，名字部分用*号替代
- **示例**: 
  - 张三 → 张*
  - 李四 → 李*
  - 王小明 → 王**
- **特殊情况**: 
  - 复姓（如欧阳、司马）: 欧阳峰 → 欧阳*
  - 单字名: 赵一 → 赵*

### 1.2 新疆姓名（含·分隔符）
- **规则**: 保留姓氏和第一个名字，后续名字用*号替代
- **示例**:
  - 阿不都·热合曼 → 阿不都·热**
  - 买买提·艾力 → 买买提·艾*
  - 古丽娜尔·阿不都拉 → 古丽娜尔·阿****
- **说明**: 根据GB/T 45574-2025要求，需特别考虑少数民族姓名的特殊格式

### 1.3 英文姓名
- **规则**: 保留首字母，其余用*号替代
- **示例**:
  - John Smith → J*** S****
  - Mary Johnson → M*** J******
  - Robert → R*****

### 1.4 韩文姓名
- **规则**: 保留姓氏，名字部分用*号替代
- **示例**:
  - 김철수 → 김**
  - 이영희 → 이**
  - 박민수 → 박**

### 1.5 日文姓名
- **规则**: 保留姓氏，名字部分用*号替代
- **示例**:
  - 山田太郎 → 山田**
  - 佐藤花子 → 佐藤**
  - 鈴木一郎 → 鈴木**

### 1.6 德文姓名
- **规则**: 保留首字母，其余用*号替代
- **示例**:
  - Müller → M*****
  - Schmidt → S******
  - Weber → W****

## 2. 手机号码脱敏规则

### 2.1 中国大陆手机号
- **规则**: 保留前3位和后4位，中间用*号替代
- **格式**: 11位数字
- **示例**: 13800138000 → 138****8000
- **依据**: GB/T 45574-2025第6.2.1条 通信联系方式脱敏要求

### 2.2 国际手机号
- **规则**: 保留国家代码和后4位，中间用*号替代
- **示例**:
  - +8613800138000 → +86****8000
  - +14155552671 → +1****2671

## 3. 固定电话脱敏规则

### 3.1 国内固定电话
- **规则**: 保留区号和后4位，中间用*号替代
- **示例**:
  - 010-12345678 → 010-****5678
  - 021-87654321 → 021-****4321

### 3.2 国际固定电话
- **规则**: 保留国家代码和后4位，中间用*号替代
- **示例**:
  - +86-10-12345678 → +86-10-****5678
  - +1-212-5550123 → +1-212-****0123

## 4. 证件号码脱敏规则

### 4.1 身份证号码（中国大陆）
- **规则**: 保留前6位（行政区划代码）和后4位，中间用*号替代
- **格式**: 18位数字
- **示例**: 110101199001011234 → 110101********1234
- **依据**: GB/T 45574-2025第6.2.2条 身份标识脱敏要求

### 4.2 护照号码
- **规则**: 保留前2位和后3位，中间用*号替代
- **示例**:
  - E12345678 → E1****678
  - G87654321 → G8****321

### 4.3 驾驶证号码
- **规则**: 保留前4位和后4位，中间用*号替代
- **示例**: 441381199001011234 → 4413********1234

### 4.4 军官证号码
- **规则**: 保留前2位和后2位，中间用*号替代
- **示例**: 军字第123456号 → 军字****56号

## 5. 银行卡号脱敏规则

### 5.1 借记卡/信用卡
- **规则**: 保留前6位（BIN号）和后4位，中间用*号替代
- **示例**: 6228480402564890018 → 622848********0018
- **依据**: GB/T 45574-2025第6.2.3条 金融账户信息脱敏要求

### 5.2 存折账号
- **规则**: 保留前4位和后4位，中间用*号替代
- **示例**: 601428301010123456 → 6014********3456

## 6. 地址脱敏规则

### 6.1 中文地址
- **规则**: 保留省市区，详细地址用*号替代
- **示例**:
  - 北京市朝阳区建国门外大街1号 → 北京市朝阳区****
  - 上海市浦东新区陆家嘴环路1000号 → 上海市浦东新区****
- **依据**: GB/T 45574-2025第6.2.4条 位置信息脱敏要求

### 6.2 英文地址
- **规则**: 保留城市和州，详细地址用*号替代
- **示例**:
  - 123 Main St, New York, NY 10001 → ****, New York, NY 10001
  - 456 Oak Ave, Los Angeles, CA 90001 → ****, Los Angeles, CA 90001

### 6.3 日文地址
- **规则**: 保留都道府县，详细地址用*号替代
- **示例**:
  - 東京都渋谷区渋谷1-1-1 → 東京都渋谷区****
  - 大阪府大阪市北区梅田1-1 → 大阪府大阪市北区****

### 6.4 德文地址
- **规则**: 保留城市，详细地址用*号替代
- **示例**:
  - Musterstraße 123, 10115 Berlin → ****, 10115 Berlin
  - Hauptstraße 45, 80331 München → ****, 80331 München

## 7. 邮箱地址脱敏规则

### 7.1 通用规则
- **规则**: 保留用户名前3个字符和域名，中间用*号替代
- **示例**:
  - zhangsan@example.com → zha****@example.com
  - john.doe@company.org → joh****@company.org
  - info@test.com → inf****@test.com

### 7.2 短用户名处理
- **规则**: 用户名长度≤3时，保留第1个字符
- **示例**:
  - a@test.com → a****@test.com
  - ab@test.com → ab****@test.com

## 8. 国籍脱敏规则

### 8.1 通用规则
- **规则**: 不脱敏，直接显示完整国籍信息
- **说明**: 国籍属于公共信息，一般不涉及个人隐私
- **示例**: 中国、美国、日本、德国

## 9. 金额脱敏规则

### 9.1 人民币金额
- **规则**: 保留货币符号和单位，具体金额用*号替代
- **示例**:
  - ¥1,000.00 → ¥***
  - ￥500 → ￥***
  - 100元 → ***元

### 9.2 外币金额
- **规则**: 保留货币符号，具体金额用*号替代
- **示例**:
  - $100.50 → $***
  - €250 → €***
  - £75.25 → £***

## 10. 验证码脱敏规则

### 10.1 短信验证码
- **规则**: 全部用*号替代
- **示例**: 123456 → ******

### 10.2 图形验证码
- **规则**: 全部用*号替代
- **示例**: A1b2 → ****

## 11. 民族脱敏规则

### 11.1 通用规则
- **规则**: 不脱敏，直接显示完整民族信息
- **说明**: 民族信息属于公共属性，一般不涉及个人隐私
- **示例**: 汉族、蒙古族、回族、藏族

## 12. 性别脱敏规则

### 12.1 通用规则
- **规则**: 不脱敏，直接显示性别信息
- **说明**: 性别信息属于基本属性，一般不涉及个人隐私
- **示例**: 男、女、未知

## 13. 生日脱敏规则

### 13.1 完整生日
- **规则**: 保留年份，月份和日期用*号替代
- **示例**:
  - 1990-01-01 → 1990-**-**
  - 1985-12-25 → 1985-**-**

### 13.2 仅年份
- **规则**: 保留世纪，具体年份用*号替代
- **示例**:
  - 1990 → 19**
  - 2000 → 20**

## 14. 其他敏感信息脱敏规则

### 14.1 IP地址
- **规则**: 保留前两段，后两段用*号替代
- **示例**: 192.168.1.100 → 192.168.*.*

### 14.2 MAC地址
- **规则**: 保留前3组，后3组用*号替代
- **示例**: 00:1A:2B:3C:4D:5E → 00:1A:2B:**:**:**

### 14.3 车牌号码
- **规则**: 保留省份简称和字母，数字用*号替代
- **示例**: 京A12345 → 京A*****

## 15. GB/T 45574-2025标准要求

### 15.1 敏感个人信息界定
根据GB/T 45574-2025标准，敏感个人信息包括但不限于：
- 身份证件号码
- 通信联系方式
- 住址信息
- 金融账户信息
- 位置信息
- 生物识别信息
- 特定身份信息

### 15.2 脱敏处理原则
- **必要性原则**: 只处理必要的敏感信息
- **最小化原则**: 脱敏后仍能满足业务需求的最小信息量
- **不可逆原则**: 脱敏后的数据不能通过技术手段还原
- **一致性原则**: 相同类型数据采用统一脱敏规则

### 15.3 特殊群体保护
- **少数民族**: 需特别考虑少数民族姓名的特殊格式
- **未成年人**: 需加强未成年人个人信息的保护
- **特殊职业**: 对特定职业人群的信息需额外保护
