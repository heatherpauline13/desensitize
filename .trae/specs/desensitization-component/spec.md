# 数据脱敏组件 - 产品需求文档

## 概述

* **摘要**: 基于Spring Boot框架实现一个可独立运行的数据脱敏组件(JAR包)。组件通过YAML配置文件定义敏感数据类型及其脱敏规则（精确匹配正则、混合上下文正则、脱敏格式、掩码符号），通过注解方式标记需要脱敏的字段。支持单一类型脱敏、长文本全面扫描脱敏、以及基于Spring AI的内嵌式AI脱敏三种模式。JAR包支持命令行运行，可处理字符串输入（输出到控制台）、文档文件（长文本脱敏后保存到result目录）和表格文件（逐字段脱敏后保存到result目录）。

* **目的**: 为各类Java应用提供统一的符合国家标准的敏感数据脱敏能力，降低敏感信息泄露风险。

* **目标用户**: Java后端开发者、数据安全工程师。

## 目标

* 提供配置驱动的脱敏规则管理，支持YAML文件中定义多种敏感数据类型
* 提供基于注解的字段级脱敏能力
* **长文本脱敏使用混合上下文正则**，通过上下文约束避免误匹配
* **非数字类型（姓名、手机号、地址）采用策略模式**，配置文件两级结构（大类 → 子类型），脱敏时自动识别子类型
* 提供基于Spring AI的AI脱敏方法
* 提供可独立运行的JAR包，支持字符串、文档文件、表格文件三种输入方式

## 功能需求

### FR-1: 配置文件驱动的脱敏规则管理

支持通过YAML配置文件定义敏感数据类型。每条类型包含：

**通用字段**：
| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| typeId | String | 类型唯一标识 | 必填 |
| name | String | 类型中文名 | — |
| maskChar | String | 掩码符号 | 继承全局 * |
| maskFlag | boolean | 是否执行脱敏 | true |

**数字类型字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| regexPattern | String | 精确匹配正则 |
| mixedRegexPattern | String | 混合上下文正则 |
| maskFormat | String | 脱敏格式模板 |

**策略模式类型字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| strategyType | boolean | 是否启用策略模式 |
| subTypes | List | 子类型列表 |
| defaultMaskFormat | String | 默认脱敏格式（fallback） |
| mixedRegexPattern | String | 混合上下文正则 |

### FR-2: 脱敏格式模板

* `preserve(N,M)` — 保留前N个和后M个字符，中间替换为掩码符号
* `replace(S,L)` — 替换从S位置开始的L个字符为掩码符号
* `maskAll()` — 全部替换为掩码符号
* `nameMask()` — 姓名专用（中文：2字保留第1屏蔽第2，3+字保留前2屏蔽其余；英文：按空格分隔保留第一个单词，其余单词屏蔽；单单词姓名保留首字母其余屏蔽）
* `emailMask()` — 邮箱专用（@前展示前3位+3个*，@后完整展示；@前<3位则全部展示+3*）
* `dateMask()` — 日期专用（保留年份，月日屏蔽）
* `landlineMask()` — 固话专用（区号不隐藏，电话号码保留最后2位）
* `addressMask()` — 地址专用（中文地址仅保留省信息其余掩码；日文地址保留県信息其余掩码；韩文地址保留道/市信息其余掩码；英文/德文/法文地址保留大区信息其余掩码）
* `passportMask()` — 护照专用（保留1位字母和最后3位数字，其余屏蔽）

### FR-3: 脱敏规则详细说明

#### 1. 姓名（name — 策略模式）
- 中文姓名2字：屏蔽第2字（如 张三 → 张*）
- 中文姓名3+字：保留前2字，屏蔽其余（如 欧阳锋 → 欧阳*）
- 英文姓名（含空格）：按空格分隔，保留第一个单词，其余单词全部屏蔽（如 John Smith → John *****）
- 英文姓名（单单词）：保留首字母，其余字母屏蔽（如 Tom → T**）
- 子类型：chinese_name, english_name, xinjiang_name, korean_name, japanese_name, default_name

#### 2. 手机号码（phone — 策略模式）
- 大陆手机：11位，1开头 → preserve(3,4)（如 13800138000 → 138****8000）
- 港澳手机：8位，9开头 → preserve(2,2)（如 91234567 → 91****67）
- 台湾手机：10位，096开头 → preserve(3,3)（如 0961234567 → 096***4567）
- 子类型：mainland_phone, hk_macau_phone, taiwan_phone

#### 3. 身份证号码（独立数字类型）
- id_card：preserve(3,4) 保留前3位和后4位（如 110101199001011234 → 110***********1234）
- id_card_with_birth：preserve(3,0) 仅保留前3位（如 110101199001011234 → 110***************）

#### 4. 护照号码（passportMask）
- 保留1位字母和最后3位数字（如 E12345678 → E*****678）

#### 5. 地址（address — 策略模式 using addressMask）
- 中文地址：仅保留省信息，其余信息掩码（如 江苏省南京市鼓楼区中山路100号 → 江苏省********）
- 日文地址：仅保留県信息，其余信息掩码（如 東京都新宿区西新宿2-8-1 → 東京都*****）
- 韩文地址：保留道/市信息，其余信息掩码（如 서울특별시 강남구 → 서울특별시 ***）
- 英文地址：保留大区（逗号或空格后最后一段），其余信息掩码（如 123 Main Street, New York → *****New York）
- 德文/法文地址：参照英文地址规则，保留大区信息
- 子类型：chinese_address, english_address, japanese_address, default_address

#### 6. 车牌号码（preserve(2,2)）
- 保留地区编码（前2位）和最后2位（如 京A12345 → 京A**45）

#### 7. 固定电话（landlineMask）
- 区号不隐藏，7-8位电话号码保留最后2位（如 010-12345678 → 010-******78）

#### 8. 日期时间（dateMask）
- 保留年份，月日信息屏蔽（如 2024-05-17 → 2024-**-**）

#### 9. 电子邮箱（emailMask）
- @前字符展示前3位+3个*，@后完整展示
- @前<3位则全部展示后加3个*（如 ab@test.com → ab***@test.com）
- （如 test@example.com → tes***@example.com）

#### 10. 银行卡号（preserve(6,4)）
- 保留前6位和后4位（如 6222021234567890123 → 622202*********0123）

### FR-4: GB/T 45574-2025敏感数据类型覆盖

1. 姓名（中文、新疆少数民族、英文、韩文、日文）— 策略模式
2. 手机号码（大陆、港澳、台湾）— 策略模式
3. 固定电话（国内、国际）
4. 证件号码（身份证、身份证含出生日期、护照、驾驶证、军官证）
5. 银行卡号（借记卡/信用卡、存折账号）
6. 地址（中文、英文、日文、韩文、德文、法文）— 策略模式
7. 邮箱地址
8. 日期时间
9. IP地址
10. MAC地址
11. 车牌号码
12. 金额（人民币、外币）
13. 验证码
14. 国籍/民族/性别（maskFlag=false）

## 非功能性需求

### NFR-1: 性能要求
* 单一类型脱敏 < 1ms
* 长文本脱敏（10KB）< 100ms
* 预编译静态Pattern减少运行时开销

### NFR-2: 安全要求
* 避免ReDoS攻击
* AI脱敏API Key不得明文输出日志

### NFR-3: AI脱敏初始化要求
* AI脱敏方法 `AiDesensitizeUtil.mask()` 必须在 `initialize()` 完成后方可调用
* 调用前应通过 `AiDesensitizeUtil.isAvailable()` 检查模块是否已正确初始化
* 若未初始化即调用，应抛出 `IllegalStateException` 明确提示
