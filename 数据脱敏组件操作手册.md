# 数据脱敏组件操作手册

## 1. 项目概述

本组件基于 **Spring Boot 3.2.x** + **Spring AI 1.0.x** + **Java 17** 构建，提供配置驱动、注解驱动、AI驱动三种脱敏方式，支持编译为独立 JAR 包运行。

**版本**：1.0.0-SNAPSHOT | **GB/T 标准**：GB/T 45574-2025

---

## 2. 项目架构

### 2.1 Maven 多模块结构

```
desensitize-parent (父POM)
├── desensitize-core          # 核心脱敏引擎
│   ├── model/                # 数据模型：SensitiveTypeConfig, MaskFormat, MaskResult
│   ├── registry/             # 规则注册中心：DesensitizeRuleRegistry
│   ├── config/               # 自动配置：DesensitizeAutoConfiguration, DesensitizeConfigProperties
│   └── engine/               # 脱敏引擎：DesensitizeUtil(公用API), RegexMaskEngine
│
├── desensitize-annotation    # 注解脱敏模块
│   └── annotation/           # @Desensitize 注解 + Jackson 序列化器
│
├── desensitize-ai            # Spring AI 脱敏模块
│   └── ai/                   # AiDesensitizeUtil + AI 自动配置
│
└── desensitize-runner        # 可运行 JAR 入口
    └── runner/               # CLI 入口 + 文件处理器
```

### 2.2 核心类关系

```
DesensitizeConfigProperties  ←→  desensitize-config.yml (YAML配置绑定)
        │
        ▼
DesensitizeAutoConfiguration  ──→  DesensitizeRuleRegistry (规则注册中心)
        │                                    │
        │                            setRegistry()
        ▼                                    ▼
DesensitizeUtil (静态API)            RegexMaskEngine
   ├── mask(content, typeId)           ├── applyMask()
   ├── maskLongText(content)           ├── applyPreserve()
   └── MaskMatch (内部类)              └── applyReplace()

AiAutoConfiguration ──→ AiDesensitizeUtil.mask(content)
                              │
                              └── ChatClient (Spring AI)
```

### 2.3 数据模型

#### SensitiveTypeConfig（敏感类型配置）

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `typeId` | String | 类型唯一标识，如 `phone`、`id_card` | **必填** |
| `name` | String | 类型中文名，如 `手机号` | — |
| `regexPattern` | String | 正则匹配模式 | **必填** |
| `maskFormat` | String | 脱敏格式模板 | **必填** |
| `maskChar` | String | 掩码符号 | 继承全局 `*` |
| `maskFlag` | boolean | 是否执行脱敏 | `true` |

#### MaskFormat（脱敏格式模板语法）

| 语法 | 含义 | 示例 |
|------|------|------|
| `preserve(N,M)` | 保留前 N 后 M 个字符，中间替换 | `preserve(3,4)` |
| `replace(start,len)` | 替换从 start 起 len 个字符 | `replace(2,4)` |
| `maskAll()` | 全部替换为掩码符号 | `maskAll()` |

## 3. 配置文件

配置文件位于 `desensitize-core/src/main/resources/desensitize-config.yml`，采用 YAML 格式。

### 3.1 全局配置

```yaml
desensitize:
  mask-char: "*"                    # 全局默认掩码符号
  config-path: ""                   # 自定义配置文件路径（可选）
```

### 3.2 AI 提示词模板

```yaml
desensitize:
  ai:
    prompt-template: "你是一个数据安全专家。请对以下文本中的敏感个人信息进行脱敏处理..."
```

> **注意**：AI 接口连接参数（API Key、Endpoint 等）在 `application.yml` 中按 Spring AI 标准配置。

### 3.3 Spring AI 连接配置示例（application.yml）

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      base-url: https://api.openai.com
      chat:
        options:
          model: gpt-4o-mini
          temperature: 0.1
```

### 3.4 敏感类型配置示例

```yaml
desensitize:
  types:
    - typeId: "chinese_name"
      name: "中文姓名"
      regexPattern: "[\u4e00-\u9fff]{2,4}"
      maskFormat: "preserve(1,0)"

    - typeId: "phone"
      name: "中国大陆手机号"
      regexPattern: "1[3-9]\\d{9}"
      maskFormat: "preserve(3,4)"

    - typeId: "id_card"
      name: "身份证号码"
      regexPattern: "\\d{17}[\\dXx]"
      maskFormat: "preserve(6,4)"

    - typeId: "bank_card"
      name: "银行卡号"
      regexPattern: "\\d{16,19}"
      maskFormat: "preserve(6,4)"

    - typeId: "nationality"
      name: "国籍"
      regexPattern: "[\u4e00-\u9fff]{2,10}"
      maskFormat: "maskAll()"
      maskFlag: true                  # true → 脱敏为 **

    - typeId: "ethnicity"
      name: "民族"
      regexPattern: "[\u4e00-\u9fff]{1,3}族"
      maskFormat: "maskAll()"
      maskFlag: false                 # false → 不脱敏
```

### 3.5 默认敏感数据类型一览

默认配置文件内置 **30 种**敏感数据类型，覆盖以下大类：

| 大类 | 类型标识（typeId） |
|------|-------------------|
| 姓名 | `chinese_name`, `xinjiang_name`, `english_name`, `korean_name`, `japanese_name`, `german_name` |
| 手机号 | `phone`, `international_phone` |
| 固话 | `landline`, `international_landline` |
| 证件 | `id_card`, `passport`, `driver_license`, `military_id` |
| 银行卡 | `bank_card`, `deposit_book` |
| 地址 | `chinese_address`, `english_address`, `japanese_address`, `german_address` |
| 邮箱 | `email` |
| 生日 | `birthday` |
| 网络标识 | `ip_address`, `mac_address`, `license_plate` |
| 金额 | `rmb_amount`, `foreign_amount` |
| 验证码 | `verification_code` |
| 基础属性 | `nationality`（maskFlag=true）, `ethnicity`（maskFlag=false）, `gender`（maskFlag=false） |

## 4. 注解使用方式

### 4.1 @Desensitize 注解

```java
import com.desensitize.annotation.Desensitize;

public class UserInfo {

    @Desensitize(type = "chinese_name")
    private String name;              // 张三 → 张*

    @Desensitize(type = "phone")
    private String phone;             // 13800138000 → 138****8000

    @Desensitize(type = "id_card")
    private String idCard;

    @Desensitize(type = "bank_card")
    private String bankCard;

    private String remark;            // 未标注 → 不脱敏

    // getters & setters ...
}
```

### 4.2 Jackson 序列化效果

```java
UserInfo user = new UserInfo();
user.setName("张三");
user.setPhone("13800138000");
user.setIdCard("110101199001011234");

ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(user);

// 输出：
// {"name":"张*","phone":"138****8000","idCard":"110101********1234","bankCard":null,"remark":null}
```

## 5. 编程 API 使用方式

### 5.1 单一类型脱敏

```java
import com.desensitize.core.engine.DesensitizeUtil;

// 中文姓名：保留姓氏
String result = DesensitizeUtil.mask("张三", "chinese_name");
// → "张*"

// 手机号：保留前3后4
result = DesensitizeUtil.mask("13800138000", "phone");
// → "138****8000"

// 身份证号：保留前6后4
result = DesensitizeUtil.mask("110101199001011234", "id_card");
// → "110101********1234"

// 国籍 maskFlag=true → 全部替换为 **
result = DesensitizeUtil.mask("中国", "nationality");
// → "**"

// 民族 maskFlag=false → 不脱敏
result = DesensitizeUtil.mask("汉族", "ethnicity");
// → "汉族"
```

### 5.2 长文本全面脱敏

```java
String text = "我叫张三，手机号13800138000，身份证110101199001011234，邮箱zhangsan@test.com";
String result = DesensitizeUtil.maskLongText(text);

// → "我叫张*，手机号138****8000，身份证110101********1234，邮箱zha****@test.com"
```

**重叠匹配规则**：长文本中多个正则匹配到同一区域时，匹配内容更长的规则优先脱敏，已脱敏区域不再重复处理。

### 5.3 AI 脱敏

```java
import com.desensitize.ai.AiDesensitizeUtil;

String content = "我叫张三，电话是13800138000，住在北京市朝阳区建国门外大街1号";
String result = AiDesensitizeUtil.mask(content);

// 前置条件：Spring AI 已正确配置，desensitize-config.yml 中已设置 ai.prompt-template
// 返回 AI 模型处理后的脱敏结果
```

### 5.4 检查 AI 是否可用

```java
if (AiDesensitizeUtil.isAvailable()) {
    String result = AiDesensitizeUtil.mask(content);
} else {
    // 回退到正则脱敏
    String result = DesensitizeUtil.maskLongText(content);
}
```

## 6. JAR 包命令行使用

### 6.1 编译打包

```bash
# 在项目根目录执行
mvn clean package -pl desensitize-runner -am

# 生成的 JAR 包位于
# desensitize-runner/target/desensitize-runner-1.0.0-SNAPSHOT.jar
```

### 6.2 查看帮助

```bash
java -jar desensitize-runner.jar
```

输出：

```
数据脱敏组件 - 命令行使用说明
================================
用法: java -jar desensitize-runner.jar [选项]

选项:
  --input.string=<文本>    对输入字符串进行脱敏，结果输出到控制台
  --input.file=<文件路径>  对文档文件进行脱敏，结果保存到 result/ 目录
  --input.table=<文件路径> 对表格文件进行脱敏，结果保存到 result/ 目录
  --mask.mode=<模式>       脱敏模式: long_text(默认) | single | ai
  --mask.type=<类型ID>     单一类型脱敏时指定的类型（与 --mask.mode=single 配合使用）
```

### 6.3 Demo — 字符串输入

```bash
# 长文本模式（默认）：扫描所有敏感类型
java -jar desensitize-runner.jar --input.string="张三的手机号是13800138000，身份证号110101199001011234"

# 输出：张*的手机号是138****8000，身份证号110101********1234
```

```bash
# 单一类型模式：仅脱敏手机号
java -jar desensitize-runner.jar --input.string="13800138000" --mask.mode=single --mask.type=phone

# 输出：138****8000
```

```bash
# AI 模式：使用 AI 接口脱敏（需配置 Spring AI）
java -jar desensitize-runner.jar --input.string="张三的电话是13800138000" --mask.mode=ai

# 输出：AI 处理后的脱敏结果
```

### 6.4 Demo — 文档文件输入

**输入文件** `test.txt`：

```
员工张三，电话13800138000，身份证110101199001011234，银行卡6228480402564890018
邮箱zhangsan@example.com，地址北京市朝阳区建国门外大街1号
```

**执行命令**：

```bash
java -jar desensitize-runner.jar --input.file=./test.txt
```

**输出** `result/test_masked.txt`：

```
员工张*，电话138****8000，身份证110101********1234，银行卡622848********0018
邮箱zha****@example.com，地址北京市朝阳区****
```

支持的文件格式：`txt`, `md`, `json`, `xml`, `csv`

### 6.5 Demo — 表格文件输入（CSV）

**输入文件** `data.csv`：

```csv
姓名,手机号,身份证号,银行卡号,地址,国家,备注
张三,13800138000,110101199001011234,6228480402564890018,北京市朝阳区建国门外大街1号,中国,VIP客户
李四,13912345678,310101198502022345,6222021001123456789,上海市浦东新区陆家嘴环路1000号,中国,普通客户
```

**执行命令**：

```bash
java -jar desensitize-runner.jar --input.table=./data.csv
```

**输出** `result/data_masked.csv`：

```csv
姓名,手机号,身份证号,银行卡号,地址,国家,备注
张*,138****8000,110101********1234,622848********0018,北京市朝阳区****,**,VIP***(脱敏)
李*,139****5678,310101********2345,622202********6789,上海市浦东新区****,**,普通**(脱敏)
```

**列处理规则**：
- 表头匹配已知类型（姓名/手机号/身份证号/银行卡号/地址/国家，支持中英文）→ 整列用单一类型脱敏
- 表头不匹配（如"备注"）→ 该列每单元格用长文本脱敏
- 表头匹配支持去空格、忽略大小写

### 6.6 Demo — 表格文件输入（XLSX）

```bash
java -jar desensitize-runner.jar --input.table=./data.xlsx
```

支持 `.xlsx` 格式，处理逻辑与 CSV 一致。

### 6.7 错误处理

```bash
# 文件不存在
java -jar desensitize-runner.jar --input.file=./notexist.txt
# → 错误: 文件不存在 - .\notexist.txt （退出码: 1）

# 多个输入模式冲突
java -jar desensitize-runner.jar --input.string="test" --input.file=./test.txt
# → 错误: --input.string、--input.file、--input.table 不能同时指定多个 （退出码: 1）

# 不支持的文件格式
java -jar desensitize-runner.jar --input.file=./image.png
# → 错误: 不支持的文件格式 .png，支持的格式: [txt, md, json, xml, csv] （退出码: 1）
```

## 7. 扩展指南

### 7.1 新增敏感数据类型

在 `desensitize-config.yml` 中添加新类型即可，无需修改代码：

```yaml
desensitize:
  types:
    - typeId: "wechat_id"
      name: "微信号"
      regexPattern: "wxid_[a-z0-9]+"
      maskFormat: "preserve(5,0)"
```

### 7.2 自定义掩码符号

```yaml
# 全局修改
desensitize:
  mask-char: "#"

# 或单个类型修改
desensitize:
  types:
    - typeId: "phone"
      maskChar: "#"
```

### 7.3 控制某类型是否脱敏

```yaml
desensitize:
  types:
    - typeId: "nationality"
      maskFlag: false    # 不脱敏，直接返回原值
```

### 7.4 切换 AI 模型

修改 `application.yml` 中的 Spring AI 配置即可：

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
      chat:
        options:
          model: gpt-4o          # 切换模型
```

## 9. 常见问题

**Q: 启动报 "DesensitizeRuleRegistry 未初始化"？**
A: 确保 `desensitize-config.yml` 文件位于 classpath 下，且格式正确。

**Q: AI 脱敏不可用？**
A: 检查 `application.yml` 中是否正确配置了 `spring.ai.openai.api-key`，且 `desensitize-config.yml` 中有 `ai.prompt-template`。

**Q: 长文本脱敏后内容错乱？**
A: 检查各类型的正则模式是否存在过于宽泛的匹配（如 `\d{4,6}` 可能误匹配），通过 `regexPattern` 增加上下文约束。
